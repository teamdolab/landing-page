# 대선포커 게임 시스템

## 📋 개요

송출용 화면과 게임 진행용 페이지가 **실시간으로 동기화**되는 게임 시스템입니다.

---

## 🎯 주요 기능

### 1. 송출용 화면 (`/game/display`)
- 관객 및 플레이어가 보는 스크린
- 실시간 게임 상태 표시
- 플레이어 정보 (점수, 후보자, 선 플레이어, 공개 카드)
- 커뮤니티 카드 표시
- 타이머 및 현재 페이즈 표시

### 2. 게임 진행용 페이지 (`/game/control`)
- 운영자가 조작하는 관리 페이지
- 게임 세션 생성
- 11단계 게임 플로우 진행
- 플레이어 액션 기록
- 점수 계산 및 관리

### 3. 실시간 동기화
- Supabase Realtime 사용
- 게임 진행 → 송출 화면 즉시 반영
- 별도 새로고침 불필요

## 🎮 게임 룰 (대선포커)

### 기본 정보
- **라운드**: 총 4라운드
- **카드**: 2-10 (4개 무늬) = 36장
- **플레이어**: 8-12명
- **족보**: 스트레이트플러쉬 > 포카드 > **플러쉬** > 풀하우스 > 스트레이트 > 트리플 > 투페어 > 원페어
  - ⚠️ **중요**: 플러쉬가 풀하우스보다 높음!

### 라운드 진행 (11단계)

1. **선 정하기**
   - 1라운드: 카드 뽑기
   - 2-4라운드: 이전 선 플레이어 +3 위치

2. **카드 딜링 (프리플랍)**
   - 선 플레이어부터 2장씩

3. **플랍 오픈**
   - 번 카드 1장 공개
   - 커뮤니티 카드 3장 오픈

4. **전략회의 I**
   - 8분 타이머

5. **출마 선언**
   - 선 플레이어부터 순서대로
   - 각 20초: 출마 or 포기

6. **후보자 연설**
   - 출마한 플레이어만
   - 각 20초: 본인 카드 2장 공개

7. **턴 오픈**
   - 번 카드 1장 공개
   - 커뮤니티 카드 1장 추가 (총 4장)

8. **전략회의 II**
   - 8분 타이머

9. **유권자 투표**
   - 포기한 플레이어만
   - 각 20초: 후보자에게 투표 or 기권

10. **리버 오픈**
    - 번 카드 1장 공개
    - 최종 커뮤니티 카드 1장 (총 5장)

11. **결과 계산 & 라운드 종료**
    - 족보 판정
    - 점수 계산
    - 다음 라운드 또는 게임 종료

### 점수 계산 규칙

#### 후보자
- **최고 족보 (단독)**: +후보자 수
- **최고 족보 (N명 동률)**: +(후보자 수 / N, 소수점 버림)
- **패배**: -후보자 수
- **득표**: +득표 수

#### 유권자 (포기자)
- **투표한 후보가 최고 족보**: +후보자 수
- **아니면**: 0점

### 최종 우승
- 4라운드 종료 후 최고 점수
- **단독 우승**: 공동 우승자 1명 지목 가능 (최하위 제외)

## 🚀 Supabase 설정

### 1단계: 게임 스키마 생성

Supabase SQL Editor에서 실행:

```sql
-- supabase-game-schema.sql 전체 복사 → 붙여넣기 → Run
```

생성되는 테이블:
- `game_sessions` - 게임 세션 관리
- `game_state` - 실시간 게임 상태
- `game_players` - 플레이어 정보 및 점수

### 2단계: Realtime 활성화

Supabase 대시보드에서:
1. Database → Replication 메뉴
2. `game_state`, `game_players` 테이블에 Realtime 활성화

## 📊 데이터 구조

### game_state (게임 상태)
```typescript
{
  round: 1-4,
  step: 1-11,
  phase: '출마 선언', '전략회의 I' 등,
  first_player: 1-12,
  current_player: 1-12,
  timer_seconds: 0-480,
  timer_active: true/false,
  community_cards: ['S2', 'D5', 'H9'],
  deck_remaining: [...],
}
```

### game_players (플레이어)
```typescript
{
  player_number: 1-12,
  hand_cards: ['S7', 'H10'],
  revealed_cards: ['S7', 'H10'],
  status: 'run' | 'giveup' | null,
  vote_to: 1-12 | 0 (기권),
  total_score: 누적 점수,
  round_score: 이번 라운드 점수,
  is_first_player: true/false,
}
```

## 🎨 송출 화면 개선 사항

### 플레이어 표시
- ✅ **점수** - 플레이어 박스 아래 표시
- ✅ **후보자** - 🎯 배지 + 테두리 강조 + 애니메이션
- ✅ **선 플레이어** - ⭐ 배지 (회전 애니메이션)
- ✅ **공개 카드** - 플레이어 아래 미니 카드 2장
- ✅ **라운드 점수 변화** - +3, -2 등 (초록/빨강)

### 레이아웃
- ✅ **라운드 단계** - 좌측 상단 1줄 (`ROUND 1 | 출마 선언`)
- ✅ **타이머** - 중앙 (전략회의 시에만 표시)
- ✅ **핸드 랭킹** - 우측 상단
- ✅ **커뮤니티 카드** - 테이블 중앙

## 🧪 테스트 방법

### 1. Supabase 스키마 적용
```bash
1. supabase-game-schema.sql 실행
2. Realtime 활성화
```

### 2. 게임 생성
```bash
1. http://localhost:3000/game/control 접속
2. 날짜 선택, 인원 선택 (8-12명)
3. "세션 생성 및 입장" 클릭
```

### 3. 송출 화면 오픈
```bash
1. 새 창/탭에서 http://localhost:3000/game/display 접속
2. 실시간 동기화 확인
```

### 4. 게임 진행
```bash
1. 진행 페이지에서 단계별 진행
2. 송출 화면에서 실시간 업데이트 확인
```

## 🔧 개발자 노트

### Realtime 구독 (송출 화면)
```typescript
supabase
  .channel('game-display')
  .on('postgres_changes', 
    { event: '*', schema: 'public', table: 'game_state' },
    () => loadGameData()
  )
  .subscribe();
```

### 게임 상태 업데이트 (진행 페이지)
```typescript
await supabase
  .from('game_state')
  .update({ step: 2, phase: '카드 딜링' })
  .eq('session_id', sessionId);
```

### 점수 계산 예시
```
후보자 3명 (A, B, C)
- A: 플러쉬 (최고)
- B: 투페어
- C: 원페어

A: +3 (후보자 수) + 득표 수
B: -3 (후보자 수) + 득표 수
C: -3 (후보자 수) + 득표 수

유권자가 A에게 투표: +3점
유권자가 B에게 투표: 0점
```
